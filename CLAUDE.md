# Celbio 開発ルール

## ビジネス概要
- セリビオ株式会社が運営する建材受発注プラットフォーム（ソーラー中心）
- 4ロール構成:
  - admin: セリビオ本部（全体管理・マージン設定・入金消込・メーカー支払い）
  - member: 加盟店（工務店）。案件登録→見積依頼→前入金→発注→納品確認
  - partner: メーカー（仕入先）。見積回答→受注確認→出荷→請求書発行
  - electrician（UI表示: 施工パートナー）: 工事会社。エリアベース施工管理→完了報告→請求書発行
- 業務フロー:
  1. 工務店が案件登録・見積依頼
  2. セリビオがメーカーに転送、メーカーが原価で回答
  3. セリビオがマージンを乗せて工務店に提示
  4. 工務店が承認 → Order(draft)作成
  5. 工務店が前入金 → セリビオが入金確認 → 発注確定（メーカーへ発注）
  6. メーカーが出荷 → 納品
  7. 施工パートナーがエリアベースで施工
  8. メーカー・施工パートナーがセリビオに請求書発行
  9. セリビオがメーカー・施工パートナーに支払い

## 価格ルール（絶対厳守）
- 工務店（member）は原価を絶対に見れない → memberUnitPrice/memberSubtotalのみ表示
- メーカー（partner）はマージン込み価格を見れない → 原価のみ表示
- 管理者（admin）は原価とマージン込み両方を見れる
- API層で価格隠蔽を実施（UI側だけでは不十分）

## マージン計算
- 計算式: `ROUNDUP(原価 / costRatio, roundingUnit) + fixedAddition`
- 適用優先順位: 品目 → カテゴリ → パートナー×カテゴリ → デフォルト
- ロジック: `lib/margin.ts`

## ロール名の対応

|-------------|---------|----------|
| admin | /admin | セリビオ管理者 |
| member | /member | 加盟店（工務店） |
| partner | /partner | メーカー |
| electrician | /electrician | 施工パートナー |

## 品質基準
- 付け焼き刃で作らない。システムとして本質的な構造にする
- 1つのファイルに複数の責務を混ぜない（UI・API・ロジック・データ変換は分離する）
- コンポーネントは表示に専念。ビジネスロジックは `lib/` に切り出す
- API呼び出しはコンポーネントに直書きしない

## コード規約
- TypeScript strict。型を省略しない
- ファイルが300行を超えたら分割を検討する
- 同じ処理を2箇所以上に書かない（共通化する）
- モバイルファースト: カードベースUI、min-h-12タッチターゲット、フルワイドボタン

## 構造
- `app/` — ページ・APIルート
- `components/` — UIコンポーネント（表示のみ）
- `lib/` — ビジネスロジック・ユーティリティ
- `prisma/` — DBスキーマ

## デプロイ
- Vercel（mainにpush → 自動デプロイ）
- Prismaスキーマ変更時はマイグレーション必須
